Spatial Mapping Using Time-of-Flight

The Spatial Mapping Using Time-of-Flight project is an integrated spatial scanning system designed to visually re-map an enclosed space using LIDAR Sensor technology. The system is comprised of a TI Microcontroller, stepper motor, ToF Sensor, and a cardboard body. The ToF sensor emits a laser of infrared light, measuring the time it takes for the pulse of light to reflect to the sensor. Using the time measurement, the sensor calculates the distance data and transmits it to the MCU using the I2C communication protocol. The product records 32 distance measurement readings approximately every 11.25째 on a circulating 360째 axis around the y-z plane. Once the data is transmitted from the MCU to a connected PC using UART communication protocol, each measurement is converted into (x,y,z) coordinates in a Python script. A model of the space is then drawn using the Open3D Python Library.




To measure the distance of objects or surfaces around the device, the VL53L1X ToF Sensor takes samples of measurements every 11.25째 of the stepper motor's full rotation. This allows the sensor to collect a total of 32 distance measurements throughout the full 360째 plane revolving around the x-axis. The sensor emits pulses of 940nm wavelength infrared light which is then reflected off neighboring surfaces back to the detector. The sensor then calculates the distance by measuring the time it takes for the laser's full trip. The distance data is then transmitted to the microcontroller using the I2C serial synchronous communication protocol.




Once the data collection process is complete, the data points must be mapped in 3D for visualization of the surrounding area. When the microcontroller's program finishes its data collection state, the Python script continues to compile the data and write it into a .xyz file. This file is then read by the Open3D python library and put into the pcd. The data collection Python script needs to be run using Python 3.9 or earlier due to the Open3D library's lack of support for newer versions. The LineSet() function is then called to map the lines to 3D coordinate vertices collected by the program. Depending on the number of scans chosen by the user, the number of coordinates in the x-direction varies. The draw_geometries() function is then used on the lineset and the visualization appears which can be used for observation and/or manipulation by the user.
